Plano de Desenvolvimento da Aplicação para Editora Tocadiscos
O plano está organizado em três categorias principais: Base de Dados (foco no armazenamento e manipulação de dados usando CSV, adaptando recursos como FMA para dados iniciais e JSON/pickle para suporte), Backend (lógica de negócio, cálculos e funcionalidades core), e Frontend (interface CLI com menu navegável). Cada categoria é dividida em tarefas sequenciais e paralelizáveis, com dependências indicadas para permitir trabalho em grupo eficiente.

Estimativa de tempo: Assumindo um grupo de 3-4 pessoas, cada tarefa pode levar 4-8 horas, dependendo da complexidade. Total estimado: 2-3 semanas.
Ferramentas gerais: Python 3.x, PyGame para áudio, Pandas para manipulação de CSV (se necessário), Pickle para histórico. Use Git para versionamento e colaboração.
Abordagem geral: Inicie com setup de dados (Base de Dados), desenvolva lógica core (Backend) em paralelo, e integre com interface (Frontend). Teste incrementalmente. Documente com comentários no código e um README.md.

1. Base de Dados
Foco: Gerenciar armazenamento em CSV (como especificado no enunciado), usando FMA para dados iniciais de músicas/álbuns. Adaptar JSON para metadados se necessário, e Pickle para histórico de ações.

Task 1.1: Setup inicial de dados (Dependências: Nenhuma)
Baixe e integre o dataset FMA (via GitHub). Converta metadados relevantes (artistas, álbuns, músicas, gêneros) para formato CSV compatível (e.g., autores.csv, albuns.csv, musicas.csv). Defina esquemas: autores (nome, nacionalidade, direitos_percentagem); albuns (autor, nome, genero, data_lancamento, unidades_vendidas, preco); musicas (album, nome, caminho_audio).
Responsável sugerido: Membro 1. Paralelismo: Pode ser feito independentemente.
Task 1.2: Funções de leitura/escrita CSV (Dependências: Task 1.1)
Implemente funções para carregar/descarregar dados de CSV (usando csv module ou Pandas). Garanta atualização automática ao adicionar/remover autores/álbuns. Inclua validação de dados (e.g., percentagens válidas).
Responsável sugerido: Membro 1. Paralelismo: Após setup, pode ser testado com dados mock.
Task 1.3: Integração de áudio e paths (Dependências: Task 1.1)
Organize ficheiros áudio do FMA em pastas por autor/álbum. Atualize CSV com caminhos relativos para reprodução. Teste acessibilidade.
Responsável sugerido: Membro 1. Paralelismo: Independente, mas integre com Backend para reprodução.
Task 1.4: Sistema de histórico com Pickle (Dependências: Task 1.2)
Crie estrutura para salvar estados de dados (snapshots) em ficheiros Pickle. Implemente funções para salvar ações (e.g., add/remove) e reverter (carregar snapshot anterior).
Responsável sugerido: Membro 1. Paralelismo: Pode ser desenvolvido em paralelo com leitura/escrita.

2. Backend
Foco: Lógica de negócio, incluindo cálculos, autenticação, pesquisa e reprodução. Use PyGame para áudio. Mantenha separado do frontend para testes unitários.

Task 2.1: Autenticação e acesso restrito (Dependências: Nenhuma)
Implemente verificação de senha (hardcoded ou simples hash). Funções para condicionar visibilidade de direitos editoriais e relatórios.
Responsável sugerido: Membro 2. Paralelismo: Independente, teste com mocks.
Task 2.2: Funções de listagem e exibição (Dependências: Base de Dados Task 1.2)
Crie funções para listar autores (nome, nacionalidade, álbuns, direitos - condicionado) e álbuns (nome, gênero, data, unidades, preço, músicas). Use estruturas de dados como listas/dicts para manipulação em memória.
Responsável sugerido: Membro 2. Paralelismo: Pode usar dados mock inicialmente.
Task 2.3: Cálculo de direitos e relatório tabular (Dependências: Task 2.2)
Implemente lógica para calcular direitos (percentagem * unidades_vendidas * preco). Gere relatório em formato tabular (usando strings formatadas ou tabulate library). Inclua ordenação (e.g., por autor, receita) e totais (por autor e geral). Restrinja com autenticação.
Responsável sugerido: Membro 2. Paralelismo: Em paralelo com listagem.
Task 2.4: Adicionar/Remover autores e atualização (Dependências: Base de Dados Task 1.2, Task 2.1)
Funções para criar novos autores (input manual: nome, nacionalidade, etc.) e remover (com confirmação). Atualize CSV e registre no histórico (integre com Pickle).
Responsável sugerido: Membro 3. Paralelismo: Após leitura/escrita, teste integrado.
Task 2.5: Pesquisa por autor/álbum/música (Dependências: Task 2.2)
Implemente busca simples (string matching) ou avançada (usando regex). Retorne resultados filtrados.
Responsável sugerido: Membro 3. Paralelismo: Independente após listagem.
Task 2.6: Reprodução de áudio (Dependências: Base de Dados Task 1.3)
Integre PyGame mixer para tocar ficheiros áudio por álbum/música. Funções para play/pause/stop. Lide com erros (e.g., ficheiro não encontrado).
Responsável sugerido: Membro 3. Paralelismo: Pode ser testado isoladamente com ficheiros sample.
Task 2.7: Histórico e reversão de ações (Dependências: Base de Dados Task 1.4, Tasks 2.4-2.6)
Integre com ações (add/remove/search/play). Funções para visualizar histórico e reverter (carregar Pickle snapshot).
Responsável sugerido: Membro 2. Paralelismo: Finalize após outras lógicas.

3. Frontend
Foco: Menu CLI navegável. Use input() para interações. Integre com backend via chamadas de funções. Embora recursos mencionem GUI, o enunciado especifica linha de comandos, então foque em CLI (pode adicionar GUI como extra se tempo permitir).

Task 3.1: Estrutura do menu principal (Dependências: Nenhuma)
Crie menu em loop com opções (e.g., 1-Listar autores, 2-Listar álbuns, 3-Calcular direitos, 4-Adicionar autor, 5-Remover, 6-Pesquisar, 7-Reproduzir áudio, 8-Histórico, 9-Sair). Use print() e input().
Responsável sugerido: Membro 4. Paralelismo: Independente, use mocks para backend.
Task 3.2: Integração de listagem e exibição (Dependências: Backend Task 2.2, Task 3.1)
Conecte menu a funções de listagem. Mostre outputs formatados (e.g., tabelas com print).
Responsável sugerido: Membro 4. Paralelismo: Após menu base.
Task 3.3: Integração de relatório e autenticação (Dependências: Backend Task 2.1 e 2.3, Task 3.1)
Adicione prompts de senha no menu para itens restritos. Exiba relatório tabular.
Responsável sugerido: Membro 4. Paralelismo: Em paralelo com outras integrações.
Task 3.4: Integração de add/remove/pesquisa (Dependências: Backend Tasks 2.4-2.5, Task 3.1)
Prompts para inputs manuais (e.g., nome autor). Chame funções backend e atualize dados.
Responsável sugerido: Membro 4. Paralelismo: Após menu.
Task 3.5: Integração de reprodução e histórico (Dependências: Backend Tasks 2.6-2.7, Task 3.1)
Menu para selecionar música/álbum e reproduzir. Opções para ver/reverter histórico.
Responsável sugerido: Membro 4. Paralelismo: Finalize após outras integrações.

Tarefas Gerais e Integração

Task 4.1: Testes unitários e integração (Dependências: Todas acima)
Escreva testes para cada função (usando unittest). Integre tudo em um main.py. Teste end-to-end (e.g., add autor -> calcular direitos -> reverter).
Responsável: Todo o grupo. Paralelismo: Incremental.
Task 4.2: Documentação e apresentação (Dependências: Task 4.1)
Comente código. Crie README com instruções de uso, estrutura de ficheiros e diagramas (e.g., fluxo de menu). Prepare demo para exame.
Responsável: Todo o grupo. Paralelismo: Ao longo do desenvolvimento.

Este plano permite trabalho paralelo: Membro 1 foca em dados, Membro 2/3 em backend, Membro 4 em frontend. Reuniões semanais para merge e resolução de dependências. Se precisar de ajustes, avise!
